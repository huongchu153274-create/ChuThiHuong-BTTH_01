# 🎯 JAVASCRIPT FRAMEWORKS
# **"React, Vue Hay Angular?" — Chọn Vũ Khí Cho 5 Năm Tới**

## 🎬 Hook

*Minh đã viết Todo App bằng Vanilla JS: DOM manipulation thủ công, state quản lý bằng biến global, event listeners khắp nơi. Code 500 dòng mà đã muốn điên.*

*"Bây giờ hãy tưởng tượng app có 50 trang," chị Hà nói. "Vanilla JS cho app lớn = địa ngục. Đó là lý do frameworks ra đời: React, Vue, Angular — chúng QUẢN LÝ SỰ PHỨC TẠP."*

> 💡 **Chỉ cần học 1 framework.** React = nhiều job nhất. Vue = dễ học nhất. Angular = enterprise.

---

## 🎨 BA LỰA CHỌN

### ⚛️ [React](./react/README.md)
**JavaScript Library** để xây dựng UI, được phát triển bởi Facebook (Meta).

**Đặc điểm:**
- ✅ Phổ biến nhất, cộng đồng lớn
- ✅ Flexible, có thể dùng với nhiều tools
- ✅ JSX syntax
- ✅ Huge ecosystem

**Phù hợp khi:**
- Cần flexibility cao
- Dự án lớn, phức tạp
- Team có kinh nghiệm

---

### 🟢 [Vue.js](./vue/README.md)
**Progressive Framework** để xây dựng UI, được phát triển bởi Evan You.

**Đặc điểm:**
- ✅ Dễ học nhất
- ✅ Template syntax giống HTML
- ✅ Progressive (tích hợp từng phần)
- ✅ Performance tốt

**Phù hợp khi:**
- Mới bắt đầu học framework
- Cần học nhanh
- Dự án vừa và nhỏ

---

### 🔴 [Angular](./angular/README.md)
**Full-Featured Framework** để xây dựng SPA, được phát triển bởi Google.

**Đặc điểm:**
- ✅ TypeScript-first
- ✅ Full-featured (routing, forms, HTTP built-in)
- ✅ Enterprise-ready
- ✅ Strong typing

**Phù hợp khi:**
- Dự án enterprise lớn
- Team quen TypeScript
- Cần structure rõ ràng

---

## 🤔 CHỌN FRAMEWORK NÀO?

### Nếu bạn là người mới:
**→ Chọn Vue.js** - Dễ học nhất, syntax giống HTML

### Nếu bạn muốn phổ biến nhất:
**→ Chọn React** - Nhiều job nhất, ecosystem lớn

### Nếu bạn thích TypeScript:
**→ Chọn Angular** - TypeScript-first, structure rõ ràng
